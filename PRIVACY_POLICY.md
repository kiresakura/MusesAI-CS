# Privacy Policy | 隱私權政策

**Last Updated: 2026-03-05**

This privacy policy describes how the MusesAI-CS chatbot system ("the System") collects, uses, and protects user information.

本隱私權政策說明 MusesAI-CS 客服機器人系統（以下簡稱「本系統」）如何蒐集、使用及保護使用者資訊。

---

## 1. Information We Collect | 蒐集的資訊

### 1.1 Conversation Data | 對話資料

The System collects the following data during conversations:

本系統在對話過程中蒐集以下資料：

- **User ID**: A platform-provided identifier (e.g., Messenger sender ID). We do not collect real names, phone numbers, or email addresses through the chatbot.
- **Message Content**: The text messages sent by users to the chatbot.
- **Bot Replies**: The responses generated by the System.
- **Timestamps**: The date and time of each message.
- **Intent Classification**: The detected intent category and confidence score for each message.

- **使用者 ID**：由平台提供的識別碼（如 Messenger sender ID）。本系統不會透過聊天機器人蒐集真實姓名、電話或電子郵件。
- **訊息內容**：使用者傳送給客服機器人的文字訊息。
- **機器人回覆**：本系統生成的回覆內容。
- **時間戳記**：每則訊息的日期與時間。
- **意圖分類**：每則訊息偵測到的意圖類別與信心度。

### 1.2 Error Logs | 錯誤日誌

The System logs error events including:

本系統記錄的錯誤事件包括：

- Error type and timestamp
- Associated user ID and message content (for debugging purposes)
- Error details (e.g., API timeout, rate limit)

- 錯誤類型與時間戳記
- 相關的使用者 ID 與訊息內容（供除錯使用）
- 錯誤詳情（如 API 逾時、限速等）

### 1.3 Information We Do NOT Collect | 我們不蒐集的資訊

- Real names, phone numbers, or email addresses (unless voluntarily provided in message text)
- Payment or financial information
- Location data
- Device information or cookies

- 真實姓名、電話或電子郵件（除非使用者主動在訊息中提供）
- 付款或金融資訊
- 位置資料
- 裝置資訊或 cookies

---

## 2. How We Use Information | 資訊使用方式

Collected data is used exclusively for:

蒐集的資料僅用於：

1. **Providing Customer Service**: Processing user messages and generating relevant responses based on the knowledge base.
2. **Conversation Context**: Maintaining recent conversation history (up to 10 messages per user) to provide contextually appropriate replies.
3. **System Improvement**: Analyzing error logs to identify and fix system issues.

1. **提供客服服務**：處理使用者訊息，並根據知識庫生成相關回覆。
2. **對話上下文**：維護近期對話歷史（每位使用者最多 10 則），以提供符合上下文的回覆。
3. **系統改善**：分析錯誤日誌以識別並修復系統問題。

本系統使用以下第三方 API 處理訊息：

## 4. Data Retention | 資料保留

| Data Type | Retention Period |
|---|---|
| Conversation history | Automatically deleted after **7 days** |
| Error logs | Rolling window of the most recent **500 entries** |

| 資料類型 | 保留期限 |
|---|---|
| 對話歷史 | **7 天**後自動刪除 |
| 錯誤日誌 | 滾動保留最近 **500 筆** |

All data is stored locally in a SQLite database on the server. No data is exported or sold to any third party.

所有資料均以 SQLite 資料庫形式儲存於伺服器本地。不會將任何資料匯出或出售給第三方。

---

## 5. Data Security | 資料安全

We implement the following measures to protect user data:

我們採取以下措施保護使用者資料：

- API keys are stored as environment variables, not in source code.
- SQLite WAL mode is used for concurrent-safe database operations.
- HTTPS is enforced for all external communications.
- Messenger webhook requests are verified using HMAC-SHA256 signature validation.

- API 金鑰以環境變數儲存，不寫入原始碼。
- 使用 SQLite WAL 模式確保資料庫並發操作安全。
- 所有對外通訊強制使用 HTTPS。
- Messenger Webhook 請求透過 HMAC-SHA256 簽名驗證。

## 6. User Rights | 使用者權利

Users may:

使用者得：

- **Request data deletion**: Contact us to delete all conversation history associated with your user ID.
- **Opt out**: Stop using the chatbot at any time. No further data will be collected.

- **要求刪除資料**：聯繫我們刪除與您的使用者 ID 相關的所有對話歷史。
- **選擇退出**：隨時停止使用聊天機器人，不會再蒐集任何資料。
